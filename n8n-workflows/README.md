# n8n Workflows

The seven workflows below run on n8n Cloud (free tier is sufficient). The installer imports them automatically; this README documents them in case you need to import or wire credentials manually.

## Files in this directory

| File | Purpose |
|---|---|
| `01-email-triage-outlook.json` | Polls Outlook/M365 inbox every 5 min, runs Claude triage, writes to Supabase `triage_emails` |
| `02-email-triage-gmail.json` *(generated by installer from the Outlook template)* | Same as above but Gmail trigger |
| `03-auto-nudge.json` | Schedule trigger — queries `nudge_state` for due nudges, generates draft, pings the dashboard for approval |
| `04-mark-triaged.json` | Webhook receiver — Apps Script calls this to mark an Outlook/Gmail message read after a draft is sent |
| `98-training-runner.json` | Bulk-runs the triage prompt against the training spreadsheet for accuracy tuning |
| `99-test-harness.json` | Test harness — runs canned email scenarios through the triage agent to detect regressions |

## Tokens in workflow JSON

The shipped JSONs contain `{{TOKEN}}` placeholders for everything personal or environment-specific:

| Token | Filled by installer from |
|---|---|
| `{{OWNER_NAME}}` | `config.local.json` -> owner.name |
| `{{OWNER_FIRST_NAME}}` | derived from owner.name |
| `{{OWNER_EMAIL}}` | owner.email |
| `{{OWNER_TITLE}}` | owner.title |
| `{{COMPANY_NAME}}` | company.name |
| `{{COMPANY_DOMAIN}}` | company.domain |
| `{{COMPANY_INDUSTRY}}` | company.industry |
| `{{TEAM_MEMBER_N}}` | optional — internal team list, leave blank if not used |
| `{{KEY_VENDOR_N}}` | optional — vendor names that need special handling |
| `{{N8N_CRED_OUTLOOK_OAUTH_ID}}` | n8n credential id after you connect Outlook in n8n UI |
| `{{N8N_CRED_ANTHROPIC_ID}}` | n8n credential id for Anthropic API |
| `{{N8N_CRED_OPENAI_ID}}` | optional — only if you wire a fallback OpenAI credential |
| `{{N8N_CRED_HTTP_ID}}` | n8n credential id for the Apps Script HTTP webhook (Bearer or shared secret) |
| `{{N8N_BASE_URL}}` | your n8n workspace URL |

The installer runs a token-substitution pass before importing. If you import manually, do the same substitution by hand or with `sed`.

## Manual import (fallback)

If the installer's automatic import failed:

1. In n8n -> Workflows -> Import from File.
2. Pick the JSON.
3. n8n will warn that credentials are missing.  Open each credential-using node and re-link to your own:
   - **Outlook trigger / Outlook node** -> Microsoft Outlook OAuth2 API
   - **LLM nodes** -> Anthropic API
   - **HTTP Request to Apps Script** -> Header Auth (shared secret you set in Apps Script Properties)
4. Activate the workflow.

## What each workflow does (quick read)

**01 Email Triage (Outlook / Gmail)**
- Trigger: inbox poll, unread only
- Build prompt with owner/company tokens substituted from app_config
- Call Claude with the triage prompt (`apps-script/triage-prompt.txt`)
- Parse JSON output
- Upsert into `triage_emails` via Supabase REST
- If `recommended_action == draft`, fire a dashboard notification

**03 Auto-Nudge**
- Trigger: schedule (default: weekday mornings)
- Query `nudge_state` for rows where `next_nudge_at <= now() AND paused = false`
- For each: generate nudge body via Apps Script `generateNudgeEmail`
- POST to dashboard inbox for approval (Apps Script writes a pending draft)
- Update `nudge_state.last_nudged_at` + bump `next_nudge_at`

**04 Mark Triaged**
- Webhook receiver
- Body: `{ messageId, action: "read|trash|spam" }`
- Calls the matching Outlook/Gmail action via the provider node
- Returns `{ ok: true }` to the caller

**98 Training Runner**
- Loads rows from a Google Sheet (training set)
- For each row: run the triage prompt, compare predicted category to ground-truth
- Writes a score CSV to Drive
- Used for prompt tuning, not part of the production runtime

**99 Test Harness**
- Canned email scenarios with expected categories
- Runs the triage prompt against each, diffs vs. expected
- Use after editing `triage-prompt.txt` to catch regressions

## Adding workflows for missing providers

Need a Gmail trigger? Copy `01-email-triage-outlook.json` -> rename the trigger node from `microsoftOutlookTrigger` to `gmailTrigger`, swap the credentials reference, adjust the field mapping (`from.emailAddress.address` -> `from`).
